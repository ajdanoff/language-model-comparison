# Сравнение языковых моделей: LSTM vs GRU vs distilgpt2

## 📋 Описание проекта

Этот проект сравнивает три языковые модели для обучения на текстовых данных:
- **LSTM** (Long Short-Term Memory) — экспериментальная модель
- **GRU** (Gated Recurrent Unit) — экспериментальная модель
- **distilgpt2** — предобученная модель от Hugging Face

## 🎯 Цель проекта

Сравнить модели по следующим метрикам:
1. **Perplexity** — качество языковой модели (меньше = лучше)
2. **Число параметров** — размер модели (меньше = быстрее)
3. **Время на эпоху** — скорость обучения (меньше = быстрее)

## 🚀 Этапы проекта

### Этап 1. Загрузка данных
- Загрузка датасета **WikiText-2** от Hugging Face
- Очистка и токенизация текстов
- Построение словаря и конвертация в индексы

### Этап 2. Обучение LSTM
- Создание LSTM-модели (однонаправленный, 2 слоя)
- Обучение с Adam optimizer
- Вычисление Val Perplexity

### Этап 3. Эксперимент: GRU vs LSTM
- Замена LSTM на GRU
- Обучение с теми же гиперпараметрами
- Сравнение perplexity, параметров, времени

### Этап 4. Оценка distilgpt2
- Загрузка предобученной distilgpt2
- Вычисление perplexity на тестовой выборке
- Примеры предсказаний

### Этап 5. Сравнение и выводы
- Финальная таблица всех моделей
- Примеры предсказаний на одних контекстах
- Выводы и рекомендации для продакт-менеджера

| GRU | ~0.4M | 61.6265 | 52.8523 | 214.12с |
| distilgpt2 | 82M | 1824.6849 | 1824.6849 | N/A |

### Выводы

🏆 Best by Perplexity: GRU
   • Perplexity: 61.6265
   • Почему лучше: Быстрее и почти не уступает (GRU)

⚡ Fastest: GRU
   • Время: 214.12с/эпоха
   • Преимущество: Быстрее экспериментов, меньше ресурсов

📦 Smallest: GRU
   • Параметры: 6,938,980 (6.94M)
   • Преимущество: Меньше памяти, быстрее инференс

### Рекомендация для продакт-менеджера

📊 Сценарии использования:

Сценарий                       | Рекомендация              | Почему                             
------------------------------------------------------------------------------------------
Производственная среда (качество) | distilgpt2                | Лучшее качество, предобучено       
Быстрые эксперименты           | GRU                       | Быстрее обучение, дешевле          
Мобильное приложение           | GRU/LSTM                  | Меньше параметров (~0.5M)          
Генерация текста (creative)    | distilgpt2                | Лучше качество генерации           
Ограниченные ресурсы           | GRU                       | Меньше CPU/GPU, дешевле            
Баланс качество/скорость       | LSTM                      | Лучше perplexity, приемлемо        


💡 Ключевые рекомендации:
1. ✅ distilgpt2 для продакшена (если бюджет есть):
   • Лучшее качество (Perplexity: 1824.6849)
   • Предобучено на больших данных, не нужно дообучать
   • 82M параметров = требует больше ресурсов (~100x больше ваших моделей)

2. ✅ GRU для экспериментов и разработки:
   • Быстрее LSTM на 1.03x
   • Почти не уступает по качеству (Perplexity: 61.6265)
   • ~6.93898.5M параметров = легко развернуть

3. ✅ LSTM если качество критично:
   • Лучший Perplexity среди проанализированных моделей: 148.8454
   • Лучше запоминает долгосрочные зависимости
   • Медленнее GRU, но приемлемо для небольших задач

4. 🎯 Мой выбор для старта:
   • Начать с GRU (быстрые эксперименты)
   • Если качество недостаточно → перейти на distilgpt2
   • LSTM - компромисс между качеством и скоростью

### Финальный вывод

🎯 ОБЩАЯ:

1. По качеству (Perplexity): GRU > LSTM > GRU
   • GRU: 61.6265
   • LSTM: 148.8454
   • GRU: 61.6265

2. По скорости: GRU > LSTM >> distilgpt2 (не обучали)
   • GRU: 214.12с/эпоха
   • LSTM: 221.52с/эпоха (1.03x медленнее)

3. По размеру: GRU < LSTM << distilgpt2
   • GRU: 6.94M параметров
   • LSTM: 7.17M параметров
   • distilgpt2: 81.9M параметров

💡 РЕКОМЕНДАЦИЯ:
   • Для старта: GRU (быстро, дешево, почти не уступает)
   • Для качества: distilgpt2 (лучше, но дороже)
   • Компромисс: LSTM (если нужно лучше GRU, но не хватает ресурсов на distilgpt2)
